🔄 Algorithm Deep Dive

🔄
FIFO (First In First Out)
Time Complexity: O(n×f) | Space Complexity: O(f)

The simplest page replacement algorithm. Removes the oldest page first.

🌍 Real World Example: Queue at a ticket counter — first person enters, first served.

🎯
LRU (Least Recently Used)
Time Complexity: O(n×f) | Space Complexity: O(f)

Replaces the page that was not used for the longest time.

🌍 Real World Example: Mobile apps — least recently used apps are closed first to free memory.

⭐
Optimal
Time Complexity: O(n²×f) | Space Complexity: O(f)

Replaces the page that will not be used for the longest time in the future.

🌍 Real World Example: Predictive caching (like YouTube/Netflix preloading videos based on future behavior).

⚠️
Belady's Anomaly

A situation where increasing the number of frames results in MORE page faults. This happens mainly in FIFO.

🌍 Real World Example: Adding more workers in a poorly managed system causing MORE delays instead of improving performance.

💡 Educational Insight:
This simulator demonstrates how different page replacement strategies affect system performance. The Optimal algorithm serves as a theoretical benchmark, while FIFO and LRU represent practical implementations used in real operating systems. Understanding these trade-offs is crucial for system design and performance optimization.
